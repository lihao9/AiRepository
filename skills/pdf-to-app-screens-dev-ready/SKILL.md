# PDF -> App页面需求整理（开发落地增强版）Skill

## 适用场景
当用户希望把 PDF 需求文档整理成“可直接用于 Android 开发拆解”的页面级需求时使用本 Skill。  
输出主维度必须是 App 页面（Screen），不是 PDF 页码。

## 输出目标
1. 识别并整理所有 App 页面。
2. 每个页面输出完整产品需求 + 技术落地信息。
3. 创建一个页面功能详情 `.md` 文件并写入完整结果。
4. 为每个页面生成一个可执行的开发 skill 文件（`SKILL.md`）。
5. 将逐页 skill 保存到项目层 `/.cursor/skills/` 目录下。
6. 让结果可直接用于：产品评审、研发排期、接口联调、测试用例设计与开发执行。

---

## 执行流程

### 步骤1：通读文档并识别页面
- 通读 PDF 全文，抽取所有 App 页面候选名。
- 合并同义页面名（如“我的订单”“订单列表页”）。
- 输出标准化页面清单（SCR-001...）。

### 步骤2：按页面重组需求（跨页聚合）
- 将分散在文档各处的同一页面信息聚合。
- 不按 PDF 页码输出；仅将页码作为“证据来源”。
- 对冲突描述建立“冲突记录”。

### 步骤3：页面级详细提取（产品维度）
每个页面提取：
- 页面目标
- 入口/出口
- 页面元素
- 功能点
- 状态设计（Loading/Empty/Success/Error/NoNetwork）
- 交互规则
- 输入校验
- 业务规则
- 异常处理
- 埋点建议
- 验收标准
- 原文依据
- 待确认项

### 步骤4：页面级技术落地提取（开发维度）
每个页面额外补充：

#### A. UI组件清单（Compose）
- 页面容器（Scaffold / Column / LazyColumn / Box）
- 核心组件（TextField / Button / Card / Dialog / Snackbar）
- 列表项组件（ItemCard 等）
- 复用组件建议（抽离哪些 Composable）
- 主题与资源建议（颜色/间距/文案资源化）

#### B. ViewModel状态模型（建议）
- `UiState` 字段草案（data class）
- `UiEvent`（用户事件）
- `UiEffect`（一次性事件：Toast/导航）
- 状态流转规则（事件 -> UseCase -> State）
- 并发与错误处理建议（协程、异常映射）

#### C. API DTO草案（建议）
- Request DTO（字段、类型、是否必填）
- Response DTO（字段、类型、可空性）
- Domain Model 映射建议
- 分页字段约定（page/pageSize/total/list）
- 错误码与提示文案映射建议

#### D. Android任务拆分建议
- UI任务
- ViewModel任务
- Domain UseCase任务
- Data Repository/API任务
- 测试任务（单测/集成/UI）

### 步骤5：写入页面功能详情文件（强制）
- 必须在项目工作区内创建一个汇总文件，建议路径：
  - `/.cursor/skills-output/app-pages-dev-spec.md`
- 文件内容必须包含完整的“总览 + 页面清单 + 页面详细 + 跨页面冲突 + 待确认项”。
- 文件写入完成后，再进入逐页 skill 生成流程。

### 步骤6：生成逐页可执行Skill文件（强制）
- 按页面清单逐个生成 skill 目录与 `SKILL.md`：
  - `/.cursor/skills/scr-001-<slug>/SKILL.md`
  - `/.cursor/skills/scr-002-<slug>/SKILL.md`
  - ...
- 每个页面 skill 必须包含：
  - frontmatter（`name`、`description`）
  - 页面目标与范围
  - 输入要求（设计稿/接口/字段规则）
  - 执行步骤（UI -> 状态 -> 交互 -> 数据 -> 测试）
  - 输出要求（涉及文件、验收标准、自测清单）
  - 约束（资源复用、异常处理、状态可追踪、禁止硬编码）

### 步骤7：全局一致性检查
- 流程闭环（页面跳转链是否闭合）
- 状态一致性（各页面错误/空态规范是否统一）
- 权限与角色规则冲突
- 字段与接口定义冲突
- 缺失页面识别（登录失效页、权限拒绝页、通用错误页）
- 数量一致性检查：
  - 页面清单数 == 页面详情数 == 生成的页面 skill 文件数

## 质量标准
- 输出按“App页面”组织，不得按 PDF 页码组织。
- 每个页面都需具备产品层 + 技术层信息。
- 必须创建并写入页面功能详情 `.md` 文件。
- 必须逐页生成可执行 `SKILL.md` 并保存到项目层 `/.cursor/skills/`。
- 页面清单、页面详情、页面 skill 文件数量必须一致。
- 禁止臆测；不确定内容标记“待确认”。
- 所有关键结论需提供原文依据（可含来源页码）。

## 约束与原则
- 文案统一资源化（strings）。
- 颜色/尺寸优先复用现有资源与主题。
- UI 不直接访问网络/数据库，统一经 ViewModel/UseCase。
- 状态可追踪（Loading/Success/Error/Empty）。
- 异常不可吞，需给用户可理解提示与开发可定位信息。
